## Affected Pages/Routes
- `/planner` — 今日计划页面，新增任务、编辑、AI重排

## Key Interactions to Verify
1. **Add task** on planner page — user adds new task with title, priority, estimated minutes
2. **Edit/delete task** on planner page — user can modify or remove existing tasks
3. **Auto matching** — Rust polls windows, AI matches current window to the right task
4. **Time accumulation** — actual time accumulates correctly on each task
5. **AI reschedule** — when a task is delayed, AI reorders remaining tasks by priority
6. **Persistence** — restart app, tasks and activities are still there
7. **Daily archiving** — new day creates new empty plan, yesterday's plan is archived

## Edge Cases
- Empty plan (no tasks added yet)
- Task with 0 estimated time (user input error)
- Multiple windows match same task
- No match found (activity goes to uncategorized)
- AI API call fails (graceful fallback needed)
- Machine sleeps/wakes — tracking continues correctly

## Critical Paths
- Full end-to-end: Add tasks → let tracking run for a few minutes → check time accumulation → reschedule → verify new order → restart app → verify data persists
