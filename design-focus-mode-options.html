🧘 专注模式 - 设计方案对比

因为专注模式是"操作状态"而非"浏览页面"，不应该与 Dashboard 同级

✅ 推荐方案 A

全局浮动按钮 + 侧边展开面板

┌────────────────────────────────────────┐ │ 页面内容 (Dashboard/Timeline/...) │ │ │ │ │ │ ┌────┐ │ │ │ 🧘 │ │ ← 右下角固定 │ └────┘ │ │ │ └────────────────────────────────────────┘ 点击后从右侧滑出： ┌───────────────────────┬───────────────┐ │ │ ╭────────────╮ │ │ │ │ 24:32 │ │ │ │ │ ████████ │ │ │ │ │ │ │ │ │ │ [暂停][结束] │ │ │ │ │ │ │ │ │ 今日: 3个 │ │ │ │ ╰────────────╯ │ │ │ │ └───────────────────────┴───────────────┘

✅ 优点

  • 任何页面都能快速进入
  • 不打断当前浏览上下文
  • 层级正确：工具级操作
  • 可以部分展开看倒计时

⚠️ 注意

  • 需要处理 z-index 层级
  • 避免遮挡重要内容
方案 B

侧边栏底部嵌入卡片

┌───────────────┐ │ Ξ Trace AI │ ├───────────────┤ │ ◉ Dashboard │ │ Timeline │ │ Tasks │ │ Analytics │ │ Settings │ │ │ │ │ │ │ ├───────────────┤ │ ╭───────────╮ │ ← 空闲状态显示 │ │ 🧘 开始专注│ │ 按钮卡片 │ ╰───────────╯ │ │ │ │ ╭───────────╮ │ ← 进行中显示 │ │ 🟢 15:42 │ │ 倒计时卡片 │ │ [暂停] │ │ │ ╰───────────╯ │ │ │ │ 👤 Alex Pro │ └───────────────┘

✅ 优点

  • 入口位置固定易找
  • 侧边栏空间利用
  • 始终可见倒计时
  • 不需要额外弹窗

⚠️ 注意

  • 侧边栏空间有限
  • 功能太多会拥挤
方案 C

全屏模态覆盖层

✅ 优点

  • 沉浸式体验，仪式感
  • 真正进入"专注"状态
  • 可以放更多统计信息

  • 完全打断当前工作流
  • 太重了，快速启动有负担
🏆 最终方案 D

组合方案：侧边栏嵌入 + 浮动按钮 + 迷你模式

┌───────────────┐ ┌───────────────────────────┐ │ Ξ Trace AI │ │ │ ├───────────────┤ │ │ │ ◉ Dashboard │ │ │ │ Timeline │ │ │ │ Tasks │ │ ╭──╮ │ ← 迷你浮动 │ Analytics │ │ │15│ │ 只显示倒计时 │ Settings │ │ ╰──╯ │ │ │ │ │ ├───────────────┤ └───────────────────────────┘ │ ╭───────────╮ │ │ │ 🧘 专注 │ │ ← 侧边栏：点击开始/查看详情 │ │ 24:32 │ │ │ ╰───────────╯ │ │ │ │ 👤 Alex Pro │ └───────────────┘ 点击侧边栏卡片 → 展开详情弹窗（非全屏） 拖拽浮动按钮 → 可移动位置 双击浮动按钮 → 暂停/继续

✅ 最佳体验

  • 多入口满足不同场景
  • 侧边栏：状态可见 + 启动
  • 浮动：倒计时始终可见
  • 弹窗：详情 + 统计

⚠️ 实施

  • 开发量稍大但值得
  • 需要统一状态管理

交互逻辑

  • 空闲状态：显示"🧘 开始专注"按钮，蓝底白字
  • 点击开始：弹出番茄时长选择（5/15/25/45/60分钟）+ 关联任务可选
  • 进行中：卡片变为绿色，显示实时倒计时 + 暂停按钮
  • 点击进行中卡片：弹出详情弹窗，显示今日统计、历史记录
  • 结束/暂停：弹窗内操作，记录完成数据

好处

  • ✅ 侧边栏导航保持整洁，只有5个核心页面
  • ✅ 专注模式作为"工具功能"而非"数据页面"
  • ✅ 状态始终可见，不需要专门进入页面查看
  • ✅ 弹窗详情足够放统计和设置
